## How to use this index

Each entry lists:
- **Purpose** — one-line description of what the system does
- **Source of Truth** — the authoritative DB table, file, or module
- **Details** — where the full specification lives
- **Do not recreate** — the most common mistake to avoid

Status markers: ✅ implemented · ⚠️ needs documentation · 🔜 planned (no backend yet)

## A — Authentication & Identity

### 1. Auth / Login / Registration
**Purpose:** Register and authenticate users (emp / co / edu); bcrypt passwords; post-login routing by user_type.
**Source of Truth:** `users` table · `auth.py` (`hash_password`, `verify_password`) · `index.auth.js` (`redirect()`)
**Details:** `CLAUDE.md → Authentication System` · `CLAUDE.md → Auth Gateway Rules` · `ARCHITECTURE.md §46`
**Do not recreate:** Do not add a second login form or second redirect logic. `redirect(u)` in `index.auth.js` is the single post-login routing authority.

---

### 2. JWT Token System
**Purpose:** Issue and validate signed JWT tokens for all API calls; payload carries `user_id`, `user_type`, `country_code`.
**Source of Truth:** JWT generation in `auth.py`; algorithm HS256; token stored in `localStorage.tw_jwt`
**Details:** `ARCHITECTURE.md §46`
**Do not recreate:** Do not pass `user_id` in request body — always extract from `token.get("user_id")` server-side. Do not use `X-User-Id` header — Bearer token only.

---

### 3. Public ID / tw_id System
**Purpose:** Unique identifier for every user: `{PREFIX}{COUNTRY_CODE}{10_HEX}`. U=emp, C=co, T=edu.
**Source of Truth:** `users.tw_id` (UNIQUE) · generated by `generate_tw_id(user_type, country_code)` in `auth.py`
**Details:** `CLAUDE.md → User ID Format` · `CLAUDE.md → Smart Public Profile Router Rules §9`
**Do not recreate:** Do not create per-entity ID generators. Future entity public IDs (J/P/A/V/D/E/L/Q/S) use `generate_public_id(prefix)` — no country code in the ID.

---

### 4. Smart Public Profile Router · `/u/{tw_id}`
**Purpose:** Unified public URL for all account types. Routes to employee / company / education profile based on `users.user_type` in DB.
**Source of Truth:** `users.user_type` · `get_user_info_by_tw_id(tw_id)` in `auth.py` · Smart Router route in `server.py`
**Details:** `ARCHITECTURE.md §63` · `CLAUDE.md → Smart Public Profile Router Rules`
**Do not recreate:** Do not create separate public routes like `/company-public/` or `/edu-public/`. Do not route by tw_id prefix alone — always query the DB.
**Legacy redirect (PR #386):** `/company-profile` is a legacy redirect only — it checks `user_type==="co"` → `/u/{tw_id}`; non-co users → `/home`; no JWT → `/login`. `/company-profile?id=123` → server-side 302 → `/u/{tw_id}`. See `CLAUDE.md → Smart Public Profile Router Rules §7`.

---

### 5. Session / localStorage Cache
**Purpose:** Client-side session cache. Stores user object after login.
**Source of Truth:** `localStorage.tw_user` (user object) · `localStorage.tw_jwt` (JWT token)
**Details:** `CLAUDE.md → Session Management` · `CLAUDE.md → Auth Gateway Rules §6`
**Do not recreate:** localStorage is a cache, not an authority. Do not gate security decisions on it. Do not add a second session key.

### 20c. Company Saved Candidates
**Purpose:** Private list of employee profiles saved by a company owner. Supports pipeline status tracking, notes, job linking, filtering, search, sorting, and statistics.
**Source of Truth:** `company_saved_candidates` DB table · `auth.py → save/remove/get/update/filter/stats functions` · `server.py → /company/saved-candidates/*`

**Frontend (Phase 6B):** Pipeline manage panel inside saved tab. `.co-cand-saved-card` / `.co-cand-manage-panel` / `updateSavedCandidate()` / `_applyCardUpdate()`.

Free @mentions stored in `company_post_comment_mentions` junction table (multi-mention support) — rendered as `<a href="/u/tw_id">`. Comment + mentions are saved as a single **atomic transaction** (BEGIN/COMMIT/ROLLBACK in `auth.py`) — no orphan comments on failure. `.pc-cmt-visual-reply` driven by `reply_to_comment_id`. Portal ⋮ menu, auto-resize textarea, relative Arabic time. **Author links:** avatar + name are `<a href="/u/{author_tw_id}">`. **@mention rendering priority (full-text left-to-right scan):** (1) reply-author → `<a>` when `reply_to_author_tw_id` present · (2) junction-table mentions (`mentions[]`) → `<a href="/u/tw_id">` per entry · (3) knownNames longest compound match → `<span>` only · (4) `@\S+` last resort → `<span>` · (5) plain text. **Mention dropdown:** portal `#pc-cmt-mention-menu`, candidates `{name, tw_id, avatar}`, max 6, keyboard nav, `visibility:hidden` measurement trick. **API contract:** request includes `mentioned_tw_ids: [tw_id1, tw_id2, ...]` (array); response returns `mentions: [{name, tw_id}, ...]` (array) per comment. Old comments without junction entries return `mentions: []` (backward compat). `mentioned_tw_id` singular column is legacy read-only.
**Source of Truth:** `company_post_comments` table · `company_post_comment_mentions` junction table (`comment_id FK ON DELETE CASCADE`, `mentioned_tw_id`, `UNIQUE(comment_id, mentioned_tw_id)`) · `auth.py → get_company_post_comments / create_company_post_comment(…, mentioned_tw_ids=None) / update / delete` · `GET /company/posts/{post_id}/comments` · `POST/PATCH/DELETE /company/posts/comments/{comment_id}` · `static/company/company.posts.js` (`_renderCommentBody` 6-arg, `_cmtBuildItem`, `_cmtBuildRepliesGroup`, `_cmtSetToggleState`, `_cmtReplyCountText`, `_cmtCheckCollapse`, `_cmtInitCollapseAll`, `_cmtInsertMention(ta, name, twId)`, `_cmtMentionedCandidates`, `_cmtKnownNames`) · `static/company/company.css` (`.pc-cmt-body.is-collapsed`, `.pc-cmt-more-btn`, `.pc-cmt-less-btn`, `.pc-cmt-replies-toggle`, `.pc-cmt-replies-box`, `.pc-cmt-mention-*`, `a.pc-cmt-author`, `a.pc-cmt-ava`, `a.pc-cmt-mention`)
**Details:** `CLAUDE.md → Post Comments System Rules` · `CLAUDE.md → Post Comments — Mention Atomicity Rules` · `ARCHITECTURE.md §65`
**Do not recreate:** Max depth=1 — server resolves deeper replies to root. Do NOT create a separate reply endpoint. `_renderCommentBody` has **6 args** `(bodyEl, text, mentionName, mentionTwId, knownNames, mentions)` — always pass all (null for absent). Multiple mentions stored in `company_post_comment_mentions` junction table only — never as comma-separated string in `company_post_comments`. Comment + mentions must be atomic (BEGIN/COMMIT/ROLLBACK) — no `except: pass` inside transaction. `mentioned_tw_ids` request must be validated server-side (each tw_id must exist in `users` table AND `@name` must appear in body). Replies go into `.pc-cmt-replies-box` — never inline in list. `_cmtSetToggleState` is the ONLY function that changes toggle text + box visibility. `_cmtCheckCollapse` MUST be called after every comment insertion. Author/mention links ALWAYS `/u/{tw_id}`. `innerHTML` forbidden for any API text. `_cmtMentionedCandidates[postId]` is an array `[{name, tw_id}]` (not a single object) — cleared after send.
