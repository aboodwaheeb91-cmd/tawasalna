## A вЂ” Authentication & Identity

### 1. Auth / Login / Registration
**Purpose:** Register and authenticate users (emp / co / edu); bcrypt passwords; post-login routing by user_type.
**Source of Truth:** `users` table В· `auth.py` (`hash_password`, `verify_password`) В· `index.auth.js` (`redirect()`)
**Details:** `CLAUDE.md в†’ Authentication System` В· `CLAUDE.md в†’ Auth Gateway Rules` В· `ARCHITECTURE.md В§46`
**Do not recreate:** Do not add a second login form or second redirect logic. `redirect(u)` in `index.auth.js` is the single post-login routing authority.

---

### 2. JWT Token System
**Purpose:** Issue and validate signed JWT tokens for all API calls; payload carries `user_id`, `user_type`, `country_code`.
**Source of Truth:** JWT generation in `auth.py`; algorithm HS256; token stored in `localStorage.tw_jwt`
**Details:** `ARCHITECTURE.md В§46`
**Do not recreate:** Do not pass `user_id` in request body вЂ” always extract from `token.get("user_id")` server-side. Do not use `X-User-Id` header вЂ” Bearer token only.

---

### 3. Public ID / tw_id System
**Purpose:** Unique identifier for every user: `{PREFIX}{COUNTRY_CODE}{10_HEX}`. U=emp, C=co, T=edu.
**Source of Truth:** `users.tw_id` (UNIQUE) В· generated by `generate_tw_id(user_type, country_code)` in `auth.py`
**Details:** `CLAUDE.md в†’ User ID Format` В· `CLAUDE.md в†’ Smart Public Profile Router Rules В§9`
**Do not recreate:** Do not create per-entity ID generators. Future entity public IDs (J/P/A/V/D/E/L/Q/S) use `generate_public_id(prefix)` вЂ” no country code in the ID.

---

### 4. Smart Public Profile Router В· `/u/{tw_id}`
**Purpose:** Unified public URL for all account types. Routes to employee / company / education profile based on `users.user_type` in DB.
**Source of Truth:** `users.user_type` В· `get_user_info_by_tw_id(tw_id)` in `auth.py` В· Smart Router route in `server.py`
**Details:** `ARCHITECTURE.md В§63` В· `CLAUDE.md в†’ Smart Public Profile Router Rules`
**Do not recreate:** Do not create separate public routes like `/company-public/` or `/edu-public/`. Do not route by tw_id prefix alone вЂ” always query the DB.

---

### 5. Session / localStorage Cache
**Purpose:** Client-side session cache. Stores user object after login.
**Source of Truth:** `localStorage.tw_user` (user object) В· `localStorage.tw_jwt` (JWT token)
**Details:** `CLAUDE.md в†’ Session Management` В· `CLAUDE.md в†’ Auth Gateway Rules В§6`
**Do not recreate:** localStorage is a cache, not an authority. Do not gate security decisions on it. Do not add a second session key.

### 11a. Cross-Tab Auth Sync (`auth-sync.js`)
**Purpose:** Detects JWT changes across browser tabs and invalidates stale owner mode instantly. Prevents a tab that was loaded as owner from staying in owner mode after logout or account switch in another tab.
**Source of Truth:** `static/shared/auth-sync.js` В· `window.TwAuthSync.onSessionChange(cb)`
**Details:** Listens to `storage`, `pageshow` (bfcache), `visibilitychange`, `focus` events. Only fires callbacks when `tw_jwt` actually changes, except `pageshow` from bfcache which forces revalidation because the page may be stale. Profile V2 wiring: `profile-v2.render.js` (bottom). Company wiring: `static/company/company.main.js`.
**Do not recreate:** Do not add per-page `storage` event listeners for session sync вЂ” use `TwAuthSync.onSessionChange`. Do not re-implement the event listener set in any page file.
